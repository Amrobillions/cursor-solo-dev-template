# Contributing

Contributions are welcome. This template exists to help solo devs ship with Cursor — your ideas and improvements make it better for everyone.

## How to contribute

- **Open an issue** — Bug in the template, unclear docs, or an idea (e.g. new rule, script, or doc).
- **Open a pull request** — Fix a typo, add a rule, improve README or FRAMEWORK, add a small script.

## What we welcome

- Fixes and clarifications to FRAMEWORK.md, README, or Cursor rules.
- New `.mdc` rules that fit the “one concern per file” approach.
- Scripts (e.g. `scripts/` .bat or cross-platform) that stay minimal and optional.
- Docs: better “first 5 minutes” guide, or short examples (without turning the repo into a full app).
- Suggestions for repo name, description, or GitHub topics so others can find the template.

## What we keep small

- No heavy tooling or mandatory CI — the template should stay “copy and go.”
- No framework-specific code in `src/` — that’s for the person who uses the template.
- Rule changes should stay aligned with the philosophy in FRAMEWORK.md (solo dev, Cursor-first, ship or kill).

## Before you submit

- Keep PRs focused (one concern per PR when possible).
- If you add or change a Cursor rule, mention it in the PR description so others can try it.

Thanks for helping the community ship.
