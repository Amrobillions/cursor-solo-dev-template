# GitHub publish checklist

**Keep this file.** It’s the reference for how this repo was published and for anyone who forks or reuses the template.

When you create the public repo, use these so others can find and use the template.

---

## 1. Repository name (pick one)

| Name | Why |
|------|-----|
| **cursor-solo-dev-template** | Clear, searchable: “cursor” + “solo dev” + “template”. |
| **solo-dev-cursor-template** | Same idea, “solo dev” first. |
| **vibe-coding-template** | Matches “VIBE CODING” in FRAMEWORK.md; memorable. |
| **cursor-project-starter** | People often search “cursor” + “starter” or “boilerplate”. |

**Recommendation:** `cursor-solo-dev-template` — best balance of clarity and search terms.

---

## 2. Repository description (short — shows in search)

Use one of these (or tweak):

- **Short:** `Cursor-first solo dev project template. Rules, memory, STATUS, DONE scope. Ship or kill.`
- **Alternative:** `Starter template for solo devs using Cursor: .cursor/rules, memory/, STATUS.md, DONE.md. MIT.`

---

## 3. Topics (tags) — so people find it

Add these in **About** → **Topics**:

```
cursor
cursor-ide
template
solo-dev
productivity
workflow
cursor-rules
ai-coding
starter
boilerplate
```

Use at least: `cursor`, `template`, `solo-dev`, `workflow`.

---

## 4. Template repository

- **Settings** → **General** → check **Template repository**.
- Then the **Use this template** button appears and people can create new repos from it.

---

## 5. Optional: README badges

You can add at the top of README (after the title):

```markdown
[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
```

---

## 6. First public push

1. Create a new repo on GitHub with the name you chose.
2. Add description and topics (steps 2 and 3 above).
3. Enable **Template repository** (step 4).
4. Push this folder (or connect existing local repo).
5. In README, you can add your GitHub username or “Maintained by @yourname” if you want.

After that, the template is public and easy to find when people search **cursor template** or **solo dev template**.
